# Pascal-S 编译器项目开发 Todo 清单（长期维护版）

## 0. 文档说明

### 0.1 目标

本清单用于统一跟踪本组课程设计从需求到验收的全部工作，避免开发周期变长后出现前后矛盾、任务遗漏、责任不清和交付口径不一致的问题。

### 0.2 使用规则

1. 本文档作为项目执行主清单，任务增删改都在此记录。
2. 每次完成任务时必须更新“状态、完成日期、证据链接”。
3. 每次新增需求或改动方案时，必须先在“决策记录”登记后再执行。
4. 每周至少一次全量复盘，检查任务依赖、风险和时间偏差。
5. 如出现阻塞，先在“问题与阻塞记录”登记，再分配处理人和截止时间。

### 0.3 状态定义

- 未开始：尚未开工。
- 进行中：已开工但未达到验收条件。
- 已完成：满足验收标准并有可验证证据。
- 阻塞：存在外部依赖或关键问题无法继续。
- 已取消：经组内确认后不再执行。

### 0.4 版本信息

- 当前版本：v1.0

- 维护方式：动态更新

## 1. 项目固定约束（不可偏离）

1. 可执行编译器文件名必须为 pascc。
2. 评测环境要求中，pascc 需要放在 /data/workspace/myshixun/bin。
3. 编译器命令行必须支持 -i filename.pas。
4. 输出文件必须是同名 .c，且位于输入 .pas 同目录。
5. Pascal Real 映射为 C float。
6. 最终目标是通过 open_set 与 hidden_set 的评测流程。
