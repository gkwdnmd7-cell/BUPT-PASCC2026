本报告用于持续记录各模块测试执行情况与分析结论，后续随着开发推进动态补充。

## 1. 词法分析模块测试分析（已完成）

### 1.1 测试目的

1. 验证词法阶段在当前里程碑下的可运行性。
2. 验证自动化测试是否通过，确认基础回归能力已建立。
3. 验证命令行链路下，合法输入与非法输入的返回行为是否符合预期。
4. 验证词法错误输出是否包含错误码、行列号和错误描述。

### 1.2 测试环境与命令

#### 构建与测试命令

1. `cmake -S . -B build`
2. `cmake --build build --config Debug`
3. `ctest --test-dir build -C Debug --output-on-failure`

#### 命令行样例验证

1. `./build/Debug/pascc.exe -i ./samples/01_features.pas`
2. `./build/Debug/pascc.exe -i ./samples/02_lex_error.pas`
3. `$LASTEXITCODE`

### 1.3 测试输入说明

1. 合法样例文件：[samples/01_features.pas](samples/01_features.pas)
2. 非法样例文件：[samples/02_lex_error.pas](samples/02_lex_error.pas)
3. 自动化词法测试程序：[tests/lexer_tests.cpp](tests/lexer_tests.cpp)

### 1.4 实际执行结果与判定

#### A. CMake 配置

结果：成功。

依据：输出包含 `Configuring done`、`Generating done`、`Build files have been written to .../build`。

判定：通过。

#### B. Debug 构建

结果：成功。

依据：

1. 生成 `lexer_tests.exe`。
2. 生成 `pascc.exe`。

判定：通过。

#### C. 自动化测试（ctest）

结果：成功。

依据：

1. `1/1 Test #1: lexer_tests ... Passed`
2. `100% tests passed, 0 tests failed out of 1`

判定：通过。

#### D. 合法样例命令行验证

命令：`pascc.exe -i ./samples/01_features.pas`

结果：成功。

依据：

1. 输出 `[INFO] Input`、`[INFO] Output`、`Tokenized items (skeleton): 37`。
2. `$LASTEXITCODE = 0`。

判定：通过。

#### E. 非法样例命令行验证

命令：`pascc.exe -i ./samples/02_lex_error.pas`

结果：按预期失败。

依据：

1. 输出了词法错误：`E105`、`E108`、`E106`、`E109`。
2. 输出汇总错误：`E199 Lexical analysis failed with 5 error(s)`。
3. `$LASTEXITCODE = 7`（词法错误退出码）。

判定：通过。

### 1.5 错误项解释（对应当前口径）

1. `E105`：标识符长度超过 10。
2. `E108`：关键字边界不满足“前后有空白或换行”的严格词法约束。
3. `E106`：出现嵌套注释（规则禁止）。
4. `E109`：字符常量非法（当前仅支持单字符，且不支持转义）。
5. `E199`：词法阶段存在错误，编译流程按策略终止。

### 1.6 覆盖性评估

已覆盖的核心点：

1. 关键字、标识符、常量与复合符号识别。
2. 注释规则（跨行支持、嵌套禁止）。
3. 错误码与定位信息输出。
4. 正常路径和错误路径的退出码行为。
5. 自动化测试链路（ctest）可重复执行。

未完全覆盖（后续补充）：

1. 更大规模边界样例（连续错误、复杂混合输入）。
2. 与语法阶段接口联调后的联合回归。

### 1.7 阶段结论

根据本次终端执行结果，词法分析模块在 M2 目标下已达到“可构建、可执行、可报错、可回归”的完成标准。
