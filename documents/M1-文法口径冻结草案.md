# M1 文法口径冻结草案（基于 1.2~1.4）

## 1. 适用范围

本草案用于统一本组 Pascal-S 编译器开发口径，依据《编译原理与技术课程设计》-202601(2).pdf 第 1.2、1.3、1.4 节整理。

## 2. 乱码与排版纠正规则

1. 文本中的乱码 `` 统一解释为空串 `ε`。
2. 文本中的全角标点（如 `；`、`，`）统一改为半角语法符号（`;`、`,`）。
3. `Digits` 与 `digits` 统一为 `digits`（不区分大小写，视为同一非终结符）。
4. 语法展示中换行断开的右侧符号串视为同一产生式右部。

## 3. 冻结版文法（开发实现基准）

```bnf
programstruct            -> program_head ; program_body .
program_head             -> program id ( idlist ) | program id
program_body             -> const_declarations var_declarations subprogram_declarations compound_statement

idlist                   -> id | idlist , id

const_declarations       -> ε | const const_declaration ;
const_declaration        -> id = const_value | const_declaration ; id = const_value
const_value              -> + num | - num | num | ' letter '

var_declarations         -> ε | var var_declaration ;
var_declaration          -> idlist : type | var_declaration ; idlist : type
type                     -> basic_type | array [ period ] of basic_type
basic_type               -> integer | real | boolean | char
period                   -> digits .. digits | period , digits .. digits

subprogram_declarations  -> ε | subprogram_declarations subprogram ;
subprogram               -> subprogram_head ; subprogram_body
subprogram_head          -> procedure id formal_parameter
                         | function id formal_parameter : basic_type
formal_parameter         -> ε | ( parameter_list )
parameter_list           -> parameter | parameter_list ; parameter
parameter                -> var_parameter | value_parameter
var_parameter            -> var value_parameter
value_parameter          -> idlist : basic_type
subprogram_body          -> const_declarations var_declarations compound_statement

compound_statement       -> begin statement_list end
statement_list           -> statement | statement_list ; statement
statement                -> ε
                         | variable assignop expression
                         | func_id assignop expression
                         | procedure_call
                         | compound_statement
                         | if expression then statement else_part
                         | for id assignop expression to expression do statement
                         | read ( variable_list )
                         | write ( expression_list )

variable_list            -> variable | variable_list , variable
variable                 -> id id_varpart
id_varpart               -> ε | [ expression_list ]
procedure_call           -> id | id ( expression_list )
else_part                -> ε | else statement
expression_list          -> expression | expression_list , expression
expression               -> simple_expression | simple_expression relop simple_expression
simple_expression        -> term | simple_expression addop term
term                     -> factor | term mulop factor
factor                   -> num
                         | variable
                         | ( expression )
                         | id ( expression_list )
                         | not factor
                         | uminus factor
```

## 4. 与词法约定联动的符号口径

1. `assignop` 固定为 `:=`。
2. `relop` 为 `=`, `<>`, `<`, `<=`, `>`, `>=`。
3. `addop` 为 `+`, `-`, `or`。
4. `mulop` 为 `*`, `/`, `div`, `mod`, `and`。
5. `id` 口径：字母开头，后接字母数字串，不区分大小写。
6. 注释口径：`{ ... }`。

## 5. 语义判定注意点（不在语法层解决）

1. `factor -> id`（或等价路径）在语法上无法区分“常量名/变量名/函数名”，由符号表与语义分析判定。
2. 函数与过程定义不允许嵌套；复合语句允许嵌套；函数/过程允许递归。
3. 传参同时支持传值与传引用。

## 6. 结论

本草案可直接作为当前开发的文法实现基线。后续若有变更，必须先更新本文件再改代码，避免口径漂移。

## 7. 已确认实现口径（2026-03-30）

1. 关键字与标识符均按不区分大小写处理；词法阶段统一转小写后识别。
2. 关键字边界按 1.4 严格执行：关键字前后需空格或换行；不满足时记为词法错误。
3. 标识符长度上限为 10 个字符；超长直接报词法错误。
4. 注释规则：支持 `{ ... }` 跨行注释，不支持嵌套；未闭合注释记为词法错误。
5. 字符常量规则：仅支持单字符，不支持转义字符。
6. `func_id` 在语法层按 `id` 进入语法结构；是否为函数标识符由语义分析和符号表判定。
7. 赋值右值和表达式中的 `id` 身份区分（常量/变量/函数）统一在语义阶段完成，语法阶段不提前分流。
8. 类型检查必须实现；类型相容与必要转换遵循课程要求并参考常见 C 编译器行为。

## 8. 实现策略补充（采纳）

1. 算术类型转换采用“整数提升 + usual arithmetic conversions”的实现策略，以保证表达式求值行为稳定且用户预期一致。
2. 数组越界策略：
    - 常量下标越界：给出 warning，不中断编译。
    - 变量下标：仅做下标类型检查，不做编译期越界判定。
3. 引用参数（var 参数）必须是可赋值左值（lvalue）；常量、字面量、一般表达式不得作为引用实参。
4. 错误恢复策略：
    - 单次编译错误上限设为 50。
    - 采用 panic mode 恢复。
    - 固定同步点按 Pascal-S 语法口径使用 `;`、`end`、`)`、`]`、`.`。
